# CloudFront

## CloudFront
```
Amazon의 CDN(Content Delivery Network) 서비스  
```

## CDN
```
지리, 물리적으로 떨어져 있는 사용자에게 .html, .css, .js 및 
이미지 파일과 같은 정적 및 동적 웹 콘텐츠를 사용자에게 
더 빨리 배포하도록 지원하는 웹 서비스
느린 응답속도 / 다운로딩 타임을 극복하는 기술
```

## 특징
```
전세계 AWS 각 리전의 엣지 로케이션에 리소스의 복사본을 미리 로드해놓고,
    사용자들이 짧은 지연 시간에 파일을 받을 수 있게 해 줌    

엣지 로케이션이라고 하는 데이터 센터의 전 세계 네트워크를 통해 콘텐츠 제공. 
    CloudFront를 통해 서비스하는 콘텐츠를 사용자가 요청하면 지연 시간이 
    가장 낮은 엣지 로케이션으로 라우팅되므로 콘텐츠 전송 성능이 뛰어남

콘텐츠가 이미 지연 시간이 가장 낮은 엣지에 있는 경우: 
    CloudFront가 콘텐츠를 즉시 제공.

콘텐츠가 엣지 로케이션에 없는 경우: 
    CloudFront는 콘텐츠의 최종 버전에 대한 소스로 지정된 오리진에서 콘텐츠 검색
    — 오리진 예: Amazon S3 버킷, MediaPackage 채널, HTTP 서버(예: 웹 서버)

```

## 장점
```
* 전송 속도 향상, EC2 인스턴스나 S3 버킷은 동작되는 장소가 리전에 한정적
    EC2 인스턴스, ELBElastic Load Balancing, S3 버킷의 내용을 
    에지 로케이션에 캐싱해서 빠른 성능
    (AWS가 아닌 사용자가 직접 구축한 외부 웹 서버의 내용도 캐싱)
* 비용 절감, S3에 직접 접근하는 것보다 저렴
* 커스텀 도메인 사용 가능
* HTTPS 사용 가능
* 각종 통계, 보안 기능 제공 
```
